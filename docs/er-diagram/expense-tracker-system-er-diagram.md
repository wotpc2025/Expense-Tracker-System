# 3.2.3 แผนภาพความสัมพันธ์ของตารางข้อมูล (ER Diagram)

แผนภาพความสัมพันธ์ของตารางข้อมูล (ER Diagram) ของโครงงานฉบับปรับปรุง แสดงโครงสร้างฐานข้อมูลหลักที่ใช้ในระบบบริหารการจัดการค่าใช้จ่าย โดยประกอบด้วย 5 เอนทิตีหลัก ได้แก่ users, budgets, expenses, admin_alerts และ admin_audit_logs ซึ่งเชื่อมโยงกันเพื่อรองรับทั้งการใช้งานของผู้ใช้ทั่วไปและการกำกับดูแลของผู้ดูแลระบบ

ในโครงสร้างล่าสุด ได้เพิ่มตาราง users เพื่อจัดเก็บข้อมูลผู้ใช้งานภายในระบบในรูปแบบ local profile โดยเชื่อมโยงกับบริการยืนยันตัวตนภายนอก (Clerk) ผ่าน clerkUserId ทำให้สามารถใช้อ้างอิงผู้ใช้งานในตารางธุรกรรมและตารางผู้ดูแลระบบได้อย่างเป็นระบบมากขึ้น

ความสัมพันธ์หลักใน ER Diagram มีดังนี้

1. users กับ budgets มีความสัมพันธ์แบบหนึ่งต่อหลาย โดยผู้ใช้หนึ่งคนสามารถสร้างงบประมาณได้หลายรายการ ผ่านฟิลด์ createdByUserId ในตาราง budgets
2. budgets กับ expenses มีความสัมพันธ์แบบหนึ่งต่อหลาย โดยงบประมาณหนึ่งรายการสามารถมีรายการค่าใช้จ่ายได้หลายรายการ ผ่านฟิลด์ budgetId ในตาราง expenses
3. users กับ expenses มีความสัมพันธ์แบบหนึ่งต่อหลาย โดยผู้ใช้หนึ่งคนสามารถสร้างรายการค่าใช้จ่ายได้หลายรายการ ผ่านฟิลด์ createdByUserId
4. users กับ admin_alerts มีความสัมพันธ์แบบหนึ่งต่อหลาย โดยผู้ดูแลระบบหนึ่งคนสามารถรับทราบเหตุการณ์แจ้งเตือนได้หลายรายการ ผ่านฟิลด์ acknowledgedByUserId
5. users กับ admin_audit_logs มีความสัมพันธ์แบบหนึ่งต่อหลาย โดยผู้ดูแลระบบหนึ่งคนสามารถมีประวัติการดำเนินการหลายเหตุการณ์ ผ่านฟิลด์ actorUserId

จากโครงสร้างดังกล่าว แสดงให้เห็นว่าฐานข้อมูลมีความครบถ้วนตามขอบเขตการทำงานของระบบ โดยครอบคลุมทั้งข้อมูลผู้ใช้งาน ข้อมูลงบประมาณ ข้อมูลค่าใช้จ่าย และข้อมูลสำหรับงานตรวจสอบในฝั่งผู้ดูแลระบบ ส่งผลให้ระบบสามารถติดตามที่มาของข้อมูลและตรวจสอบเหตุการณ์ย้อนหลังได้ชัดเจนมากยิ่งขึ้น

## เวอร์ชันสั้นสำหรับใส่ใต้รูป

ภาพที่ 3-2 แสดง ER Diagram ของระบบบริหารการจัดการค่าใช้จ่าย ซึ่งประกอบด้วย 5 ตารางหลัก ได้แก่ users, budgets, expenses, admin_alerts และ admin_audit_logs โดยมีความสัมพันธ์สำคัญแบบหนึ่งต่อหลายระหว่าง users กับตารางธุรกรรม/ตารางตรวจสอบ และระหว่าง budgets กับ expenses เพื่อรองรับการติดตามข้อมูลการใช้งานได้อย่างครบถ้วน
