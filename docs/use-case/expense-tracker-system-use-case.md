# Use Case Description - Expense Management System

## คำอธิบายแผนภาพกรณีการใช้งาน (Use Case Description)

จากแผนภาพกรณีการใช้งานของระบบ Expense Tracker System ผู้จัดทำได้กำหนดคำอธิบายของแต่ละกรณีการใช้งาน เพื่อแสดงขอบเขตการทำงานของระบบ เงื่อนไขก่อนการทำงาน ลำดับเหตุการณ์ และผลลัพธ์ที่เกิดขึ้นหลังจากระบบทำงานเสร็จสิ้น โดยมีรายละเอียดดังต่อไปนี้

### UC_01 สมัครสมาชิก

ผู้ใช้งานที่ยังไม่มีบัญชีจะเข้าสู่ระบบผ่านหน้า Landing Page ของเว็บแอปพลิเคชันก่อน เมื่อผู้ใช้งานกดปุ่ม Get Started ระบบจะนำทางไปยังหน้าสมัครสมาชิก เพื่อให้ผู้ใช้งานสร้างบัญชีสำหรับใช้งานฟังก์ชันต่าง ๆ ภายใน Expense Tracker System หลังจากผู้ใช้งานกรอกข้อมูลหรือดำเนินการลงทะเบียนผ่านระบบยืนยันตัวตนเรียบร้อยแล้ว ระบบจะสร้างบัญชีผู้ใช้งานและเตรียมความพร้อมสำหรับการเข้าสู่ระบบและใช้งานหน้าหลักของระบบต่อไป

| รายการ | รายละเอียด |
|---|---|
| Use Case ID | UC_01 |
| Use Case Name | สมัครสมาชิก |
| Brief Description | ผู้ใช้งานเริ่มจากหน้า Landing Page และไปสร้างบัญชีใหม่เพื่อใช้เข้าสู่ระบบ |
| Triggering Event | ผู้ใช้งานกดปุ่ม Get Started จากหน้า Landing Page |
| Actors | ผู้ใช้งานทั่วไป (User) |
| Preconditions | ผู้ใช้งานยังไม่มีบัญชีในระบบ |
| Postconditions | ระบบสร้างบัญชีผู้ใช้งานใหม่สำเร็จ |

**Flow of Events**

| Actor | System |
|---|---|
| 1. ผู้ใช้งานเข้าสู่หน้า Landing Page ของเว็บแอปพลิเคชัน | 2. ระบบแสดงหน้า Landing Page พร้อมปุ่ม Get Started |
| 3. ผู้ใช้งานกดปุ่ม Get Started | 4. ระบบนำทางไปยังหน้าสมัครสมาชิก |
| 5. ผู้ใช้งานกรอกข้อมูลหรือเลือกวิธีสมัครสมาชิกที่ระบบรองรับ | 6. ระบบตรวจสอบข้อมูลและประมวลผลการลงทะเบียน |
| 7. ผู้ใช้งานยืนยันการสมัครสมาชิก | 8. ระบบสร้างบัญชีผู้ใช้งานและกำหนดเส้นทางไปยังหน้าหลักของระบบ |

**Exceptions**

- หากข้อมูลไม่ครบถ้วนหรือไม่ถูกต้อง ระบบจะแจ้งเตือนให้ผู้ใช้งานแก้ไข
- หากบริการยืนยันตัวตนไม่พร้อมใช้งาน ระบบจะไม่สามารถสร้างบัญชีได้

### UC_02 เข้าสู่ระบบ

ผู้ใช้งานหรือผู้ดูแลระบบที่มีบัญชีอยู่แล้วจะเริ่มจากหน้า Landing Page เช่นเดียวกัน โดยกดปุ่ม Get Started เพื่อเข้าสู่หน้าสมัครสมาชิก จากนั้นเลือกตัวเลือกด้านล่าง Already have an account? Sign in เพื่อไปยังหน้าเข้าสู่ระบบ และเลือกวิธีการเข้าสู่ระบบตามที่เคยสมัครไว้ เมื่อระบบตรวจสอบข้อมูลการยืนยันตัวตนเรียบร้อยแล้ว จะนำผู้ใช้งานทั่วไปไปยังหน้าแดชบอร์ด และนำผู้ดูแลระบบไปยังหน้าสำหรับผู้ดูแลตามสิทธิ์ที่กำหนด

| รายการ | รายละเอียด |
|---|---|
| Use Case ID | UC_02 |
| Use Case Name | เข้าสู่ระบบ |
| Brief Description | ผู้ใช้งานเริ่มจากหน้า Landing Page แล้วเข้าสู่ระบบผ่านเส้นทาง Sign up -> Sign in |
| Triggering Event | ผู้ใช้งานกดปุ่ม Get Started และเลือก Already have an account? Sign in |
| Actors | ผู้ใช้งานทั่วไป (User), ผู้ดูแลระบบ (Admin) |
| Preconditions | ผู้ใช้งานมีบัญชีอยู่ในระบบแล้ว |
| Postconditions | ระบบสร้างสถานะการเข้าสู่ระบบและพาไปยังหน้าที่เกี่ยวข้อง |

**Flow of Events**

| Actor | System |
|---|---|
| 1. ผู้ใช้งานเข้าสู่หน้า Landing Page | 2. ระบบแสดงหน้า Landing Page พร้อมปุ่ม Get Started |
| 3. ผู้ใช้งานกดปุ่ม Get Started | 4. ระบบนำทางไปยังหน้าสมัครสมาชิก (Sign up) |
| 5. ผู้ใช้งานเลือก Already have an account? Sign in | 6. ระบบนำทางไปยังหน้าเข้าสู่ระบบ (Sign in) |
| 7. ผู้ใช้งานเลือกวิธีเข้าสู่ระบบหรือกรอกข้อมูลตามวิธีที่เคยสมัครไว้ | 8. ระบบตรวจสอบและยืนยันตัวตนของผู้ใช้งาน |
| 9. ผู้ใช้งานเข้าสู่ระบบสำเร็จ | 10. ระบบตรวจสอบบทบาทและนำทางไปยังหน้า Dashboard หรือหน้า Admin |

**Exceptions**

- หากข้อมูลเข้าสู่ระบบไม่ถูกต้อง ระบบจะปฏิเสธการเข้าใช้งาน
- หากไม่สามารถยืนยันตัวตนได้ ระบบจะยุติกระบวนการเข้าสู่ระบบ

### UC_03 ดูแดชบอร์ดและภาพรวม

ผู้ใช้งานทั่วไปสามารถดูข้อมูลสรุปทางการเงินของตนเองผ่านหน้าแดชบอร์ด ซึ่งประกอบด้วยงบประมาณ ค่าใช้จ่าย ยอดคงเหลือ และภาพรวมการใช้จ่ายตามช่วงเวลาที่เลือก ระบบจะรวบรวมข้อมูลจากฐานข้อมูลมาประมวลผลและแสดงผลในรูปแบบที่เข้าใจง่าย เช่น การ์ดสรุป ตาราง และกราฟ

| รายการ | รายละเอียด |
|---|---|
| Use Case ID | UC_03 |
| Use Case Name | ดูแดชบอร์ดและภาพรวม |
| Brief Description | ผู้ใช้งานดูภาพรวมทางการเงินของตนเอง |
| Triggering Event | ผู้ใช้งานเข้าสู่หน้า Dashboard |
| Actors | ผู้ใช้งานทั่วไป (User) |
| Preconditions | ผู้ใช้งานเข้าสู่ระบบสำเร็จ |
| Postconditions | ระบบแสดงข้อมูลสรุปงบประมาณและค่าใช้จ่าย |

**Flow of Events**

| Actor | System |
|---|---|
| 1. ผู้ใช้งานเปิดหน้า Dashboard | 2. ระบบดึงข้อมูลงบประมาณและค่าใช้จ่ายของผู้ใช้งาน |
| 3. ผู้ใช้งานเลือกช่วงเวลาที่ต้องการดูข้อมูล | 4. ระบบประมวลผลข้อมูลและแสดงผลเป็นภาพรวมบนหน้าจอ |

**Exceptions**

- หากไม่พบข้อมูล ระบบจะแสดงสถานะว่างหรือค่าเริ่มต้น
- หากดึงข้อมูลล้มเหลว ระบบจะแจ้งเตือนผู้ใช้งาน

### UC_04 สร้างข้อมูลงบประมาณ

ผู้ใช้งานสามารถสร้างงบประมาณใหม่เพื่อกำหนดกรอบการใช้จ่ายของตนเอง โดยระบบรองรับการระบุชื่องบประมาณ จำนวนเงิน ไอคอน และหมวดหมู่เริ่มต้น เมื่อบันทึกสำเร็จ ระบบจะเพิ่มรายการงบประมาณใหม่ลงฐานข้อมูลและแสดงผลในหน้าจัดการงบประมาณของผู้ใช้งาน

| รายการ | รายละเอียด |
|---|---|
| Use Case ID | UC_04 |
| Use Case Name | สร้างข้อมูลงบประมาณ |
| Brief Description | ผู้ใช้งานสร้างงบประมาณใหม่ในระบบ |
| Triggering Event | ผู้ใช้งานเลือกเพิ่มงบประมาณ |
| Actors | ผู้ใช้งานทั่วไป (User) |
| Preconditions | ผู้ใช้งานเข้าสู่ระบบสำเร็จ |
| Postconditions | ระบบบันทึกข้อมูลงบประมาณใหม่ลงฐานข้อมูล |

**Flow of Events**

| Actor | System |
|---|---|
| 1. ผู้ใช้งานเปิดหน้าจัดการงบประมาณ | 2. ระบบแสดงแบบฟอร์มสร้างงบประมาณ |
| 3. ผู้ใช้งานกรอกชื่องบประมาณ จำนวนเงิน ไอคอน และหมวดหมู่ | 4. ระบบตรวจสอบความถูกต้องของข้อมูล |
| 5. ผู้ใช้งานยืนยันการบันทึก | 6. ระบบเพิ่มข้อมูลงบประมาณและแสดงรายการล่าสุด |

**Exceptions**

- หากจำนวนเงินไม่ถูกต้อง ระบบจะไม่อนุญาตให้บันทึก
- หากข้อมูลจำเป็นไม่ครบ ระบบจะแจ้งให้ผู้ใช้งานกรอกเพิ่มเติม

### UC_05 แก้ไขและลบงบประมาณ

ผู้ใช้งานสามารถแก้ไขรายละเอียดของงบประมาณที่มีอยู่ เช่น ชื่อ จำนวนเงิน ไอคอน หรือหมวดหมู่ และสามารถลบงบประมาณที่ไม่ต้องการใช้งานแล้วได้ หากมีการลบงบประมาณ ระบบจะจัดการข้อมูลที่เกี่ยวข้องตามเงื่อนไขของระบบเพื่อให้ข้อมูลยังคงสอดคล้องกัน

| รายการ | รายละเอียด |
|---|---|
| Use Case ID | UC_05 |
| Use Case Name | แก้ไขและลบงบประมาณ |
| Brief Description | ผู้ใช้งานแก้ไขหรือลบข้อมูลงบประมาณที่มีอยู่ |
| Triggering Event | ผู้ใช้งานเลือกคำสั่งแก้ไขหรือลบงบประมาณ |
| Actors | ผู้ใช้งานทั่วไป (User) |
| Preconditions | ผู้ใช้งานมีข้อมูลงบประมาณอยู่ในระบบ |
| Postconditions | ข้อมูลงบประมาณถูกอัปเดตหรือลบออกจากระบบ |

**Flow of Events**

| Actor | System |
|---|---|
| 1. ผู้ใช้งานเลือกงบประมาณที่ต้องการจัดการ | 2. ระบบแสดงรายละเอียดงบประมาณ |
| 3. ผู้ใช้งานเลือกแก้ไขหรือเลือกลบงบประมาณ | 4. ระบบแสดงแบบฟอร์มแก้ไขหรือหน้าต่างยืนยันการลบ |
| 5. ผู้ใช้งานยืนยันการดำเนินการ | 6. ระบบอัปเดตหรือลบข้อมูลในฐานข้อมูลและรีเฟรชผลลัพธ์ |

**Exceptions**

- หากไม่พบงบประมาณที่เลือก ระบบจะไม่สามารถดำเนินการได้
- หากเกิดข้อผิดพลาดระหว่างอัปเดตหรือลบ ระบบจะแจ้งเตือนผู้ใช้งาน

### UC_07 แก้ไขและลบรายการค่าใช้จ่าย

ผู้ใช้งานสามารถแก้ไขข้อมูลของรายการค่าใช้จ่ายที่บันทึกไว้แล้ว หรือเลือกลบรายการที่ไม่ต้องการออกจากระบบได้ เมื่อการดำเนินการเสร็จสิ้น ระบบจะอัปเดตยอดการใช้จ่ายของงบประมาณที่เกี่ยวข้องให้ถูกต้องตามข้อมูลล่าสุด

| รายการ | รายละเอียด |
|---|---|
| Use Case ID | UC_07 |
| Use Case Name | แก้ไขและลบรายการค่าใช้จ่าย |
| Brief Description | ผู้ใช้งานแก้ไขหรือลบข้อมูลค่าใช้จ่ายที่มีอยู่ |
| Triggering Event | ผู้ใช้งานเลือกคำสั่งแก้ไขหรือลบรายการค่าใช้จ่าย |
| Actors | ผู้ใช้งานทั่วไป (User) |
| Preconditions | มีรายการค่าใช้จ่ายอยู่ในระบบ |
| Postconditions | รายการค่าใช้จ่ายถูกอัปเดตหรือลบ และยอดรวมถูกคำนวณใหม่ |

**Flow of Events**

| Actor | System |
|---|---|
| 1. ผู้ใช้งานเลือกค่าใช้จ่ายที่ต้องการจัดการ | 2. ระบบแสดงรายละเอียดรายการค่าใช้จ่าย |
| 3. ผู้ใช้งานเลือกแก้ไขหรือลบรายการ | 4. ระบบแสดงแบบฟอร์มหรือคำยืนยันการลบ |
| 5. ผู้ใช้งานยืนยันการดำเนินการ | 6. ระบบอัปเดตหรือลบข้อมูล และรีเฟรชยอดรวมของงบประมาณ |

**Exceptions**

- หากไม่พบรายการค่าใช้จ่ายที่เลือก ระบบจะไม่สามารถดำเนินการได้

| Actors | ผู้ดูแลระบบ (Admin) |
| Preconditions | ผู้ดูแลระบบเข้าสู่ระบบและมีสิทธิ์เข้าถึงข้อมูลบันทึกเหตุการณ์ |
| Postconditions | ระบบแสดงรายการบันทึกเหตุการณ์พร้อมรายละเอียดที่เกี่ยวข้อง |

**Flow of Events**

| Actor | System |
|---|---|
| 1. ผู้ดูแลระบบเปิดหน้าบันทึกเหตุการณ์ | 2. ระบบดึงข้อมูลเหตุการณ์จากฐานข้อมูล |
| 3. ผู้ดูแลระบบตรวจสอบรายการเหตุการณ์ | 4. ระบบแสดงข้อมูลวันเวลา ผู้กระทำ และคำอธิบายเหตุการณ์ |

**Exceptions**

- หากไม่มีข้อมูลบันทึกเหตุการณ์ ระบบจะแสดงสถานะไม่มีข้อมูล
- หากดึงข้อมูลล้มเหลว ระบบจะแจ้งเตือนผู้ดูแลระบบ
