# ข้อความบรรยายใต้รูป System Diagram

แผนภาพสถาปัตยกรรมระบบบริหารการจัดการค่าใช้จ่าย (System Diagram)

ระบบบริหารการจัดการค่าใช้จ่ายถูกออกแบบแบบเลเยอร์ (Layered Architecture) เพื่อให้เห็นลำดับการทำงานชัดเจนตั้งแต่ Client Layer, Edge & Access Layer, Application Layer, Data Layer และ External Services โดยผู้ใช้งานทั่วไปและผู้ดูแลระบบเริ่มต้นใช้งานผ่านเว็บเบราว์เซอร์ จากนั้นคำขอถูกส่งผ่าน Cloudflare Zero Trust และ Nginx Proxy Manager ก่อนเข้าสู่แอปพลิเคชันหลักที่พัฒนาด้วย Next.js

ภายใน Application Layer แบ่งหน้าที่เป็นส่วน Web UI, Server Actions, API Routes และ Business Rules เพื่อรองรับการทำงานหลักของระบบ เช่น การจัดการงบประมาณและค่าใช้จ่าย การตรวจสอบความถูกต้องของข้อมูล และการสร้างเหตุการณ์แจ้งเตือน/บันทึกการทำงานของผู้ดูแลระบบ ในส่วนของการยืนยันตัวตน ระบบเชื่อมต่อกับบริการภายนอก Clerk เพื่อจัดการกระบวนการสมัครสมาชิก เข้าสู่ระบบ และตรวจสอบสถานะเซสชันผู้ใช้งาน

สำหรับ Data Layer ข้อมูลธุรกรรมหลักถูกจัดเก็บในฐานข้อมูล MySQL ผ่าน Drizzle ORM โดยประกอบด้วยตาราง users, budgets, expenses, admin_alerts และ admin_audit_logs ทั้งนี้ตาราง users ทำหน้าที่เป็น local profile mirror ที่ซิงก์จาก Clerk เพื่อรองรับการเชื่อมโยง Foreign Key ภายในระบบ

นอกจากนี้ ระบบยังเชื่อมต่อกับบริการ OpenRouter AI สำหรับฟีเจอร์สแกนใบเสร็จ โดยแอปพลิเคชันจะส่งข้อมูลรูปภาพใบเสร็จไปประมวลผลและรับผลลัพธ์กลับมาในรูปแบบรายการค่าใช้จ่ายเพื่อให้ผู้ใช้ตรวจสอบก่อนบันทึกเข้าสู่ฐานข้อมูล

ดังนั้น แผนภาพนี้สะท้อนความสัมพันธ์ขององค์ประกอบหลักของระบบอย่างเป็นลำดับและตรวจสอบได้ ตั้งแต่การเข้าถึงระบบ การประมวลผลภายในแอปพลิเคชัน การเชื่อมต่อบริการภายนอก ไปจนถึงการจัดเก็บข้อมูลและการติดตามเหตุการณ์ฝั่งผู้ดูแลระบบ

## คำอธิบายโดยสรุป

แผนภาพนี้แสดงโครงสร้างระบบแบบเลเยอร์ของ Expense Management System โดยผู้ใช้งานเข้าผ่านเว็บเบราว์เซอร์และชั้น Edge Security ก่อนถึงแอปพลิเคชัน Next.js ซึ่งเชื่อมต่อ Clerk สำหรับยืนยันตัวตน เชื่อมต่อ MySQL ผ่าน Drizzle ORM สำหรับจัดเก็บข้อมูลหลัก และเชื่อมต่อ OpenRouter AI สำหรับประมวลผลสแกนใบเสร็จ
