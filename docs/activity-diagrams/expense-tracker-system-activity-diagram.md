# แผนภาพกิจกรรม (Activity Diagram) — ระบบ Expense Tracker

จากรายละเอียดของยูสเคสหลักทั้งหมด แผนภาพกิจกรรมใช้เพื่ออธิบายลำดับการทำงานระหว่างผู้ใช้งานและระบบในแต่ละกระบวนการ โดยเน้นจุดเริ่มต้น เงื่อนไขการตัดสินใจ การประมวลผลของระบบ และผลลัพธ์ที่ผู้ใช้งานได้รับ เพื่อให้เห็นภาพรวมการทำงานของระบบบริหารการจัดการค่าใช้จ่ายอย่างเป็นขั้นตอน

**ตำแหน่งไฟล์แผนภาพ:**
- โฟลเดอร์: `docs/activity-diagrams/expense-tracker-system/`
- รูปแบบชื่อไฟล์: `expense-tracker-system-activity-ucXX-<use-case-name>.puml`
- หมายเหตุ: ทุกไฟล์ในโฟลเดอร์นี้เป็น Activity Diagram ของระบบ Expense Tracker System

## แผนภาพกิจกรรมการสมัครสมาชิก (UC_01)

แผนภาพกิจกรรมนี้อธิบายขั้นตอนการสมัครสมาชิก เริ่มจากผู้ใช้งานเข้าสู่หน้า Landing Page กดปุ่ม Get Started และกรอกข้อมูลสมัครสมาชิก จากนั้นระบบตรวจสอบความถูกต้องของข้อมูล หากข้อมูลถูกต้องระบบจะสร้างบัญชีผู้ใช้ผ่านบริการยืนยันตัวตนและแสดงผลสำเร็จ แต่หากข้อมูลไม่ถูกต้องระบบจะแจ้งข้อผิดพลาดให้ผู้ใช้งานแก้ไข

## แผนภาพกิจกรรมการเข้าสู่ระบบ (UC_02)

แผนภาพกิจกรรมนี้แสดงขั้นตอนการเข้าสู่ระบบของผู้ใช้และผู้ดูแลระบบ โดยผู้ใช้งานกรอกอีเมลและรหัสผ่าน จากนั้นระบบตรวจสอบข้อมูลเข้าสู่ระบบและสิทธิ์การใช้งาน หากข้อมูลถูกต้องจะพาเข้าสู่หน้าหลักตามบทบาทผู้ใช้งาน แต่หากไม่ถูกต้องจะแสดงข้อความแจ้งเตือนและสิ้นสุดกระบวนการ

## แผนภาพกิจกรรมการดูแดชบอร์ดและภาพรวม (UC_03)

แผนภาพกิจกรรมนี้อธิบายการเข้าดูภาพรวมทางการเงิน โดยผู้ใช้งานเลือกหน้าแดชบอร์ด ระบบจะดึงข้อมูลงบประมาณและค่าใช้จ่าย คำนวณยอดรวมและสถิติที่เกี่ยวข้อง แล้วแสดงผลในรูปแบบการ์ดสรุปและกราฟเพื่อช่วยวิเคราะห์สถานะการใช้จ่าย

## แผนภาพกิจกรรมการสร้างข้อมูลงบประมาณ (UC_04)

แผนภาพกิจกรรมนี้แสดงกระบวนการสร้างงบประมาณใหม่ ผู้ใช้งานกรอกชื่องบประมาณ วงเงิน และหมวดหมู่ จากนั้นระบบตรวจสอบความถูกต้องของข้อมูล หากผ่านเงื่อนไขจะบันทึกข้อมูลและแสดงผลสำเร็จ แต่หากไม่ผ่านจะมีการแจ้งข้อผิดพลาด

## แผนภาพกิจกรรมการแก้ไขและลบงบประมาณ (UC_05)

แผนภาพกิจกรรมนี้แสดงการจัดการงบประมาณเดิม โดยมีจุดตัดสินใจระหว่างการแก้ไขหรือการลบ หากเลือกแก้ไขระบบจะบันทึกค่าที่เปลี่ยนแปลงและแสดงผลสำเร็จ หากเลือกการลบระบบจะลบข้อมูลงบประมาณและข้อมูลที่เกี่ยวข้องแล้วแจ้งผลการดำเนินการ

## แผนภาพกิจกรรมการเพิ่มรายการค่าใช้จ่าย (UC_06)

แผนภาพกิจกรรมนี้อธิบายการบันทึกรายจ่ายใหม่ ผู้ใช้งานกรอกชื่อรายการ จำนวนเงิน และหมวดหมู่ จากนั้นระบบตรวจสอบข้อมูลก่อนบันทึกลงฐานข้อมูล เมื่อบันทึกสำเร็จระบบจะอัปเดตยอดคงเหลือของงบประมาณให้เป็นปัจจุบัน

## แผนภาพกิจกรรมการแก้ไขและลบรายการค่าใช้จ่าย (UC_07)

แผนภาพกิจกรรมนี้แสดงการจัดการรายการค่าใช้จ่ายที่มีอยู่ โดยผู้ใช้งานสามารถเลือกแก้ไขหรือเลือกลบรายการ ระบบจะดำเนินการตามทางเลือกที่ผู้ใช้งานยืนยัน พร้อมทั้งอัปเดตข้อมูลสรุปค่าใช้จ่ายให้สอดคล้องกับรายการล่าสุด

## แผนภาพกิจกรรมการอัปโหลดใบเสร็จ (AI Scan) (UC_08)

แผนภาพกิจกรรมนี้อธิบายขั้นตอนการอัปโหลดใบเสร็จเพื่อสแกนด้วย AI เริ่มจากผู้ใช้งานเลือกไฟล์รูป จากนั้นระบบตรวจสอบประเภทและขนาดไฟล์ หากผ่านเงื่อนไขระบบจะส่งข้อมูลไปประมวลผลและแสดงผลรายการที่อ่านได้ แต่หากไม่ผ่านจะมีการแจ้งข้อผิดพลาดในการอัปโหลด

## แผนภาพกิจกรรมการยืนยันผลการสแกน (UC_09)

แผนภาพกิจกรรมนี้แสดงขั้นตอนหลังการสแกนใบเสร็จ ผู้ใช้งานตรวจสอบและแก้ไขข้อมูลรายการที่ระบบอ่านได้ก่อนยืนยันการบันทึก จากนั้นระบบตรวจสอบข้อมูลอีกครั้ง บันทึกรายการแบบหลายแถว และอัปเดตยอดค่าใช้จ่ายโดยรวม

## แผนภาพกิจกรรมการดูรายงานและสถิติ (UC_10)

แผนภาพกิจกรรมนี้อธิบายขั้นตอนการเรียกดูรายงาน ผู้ใช้งานกำหนดเงื่อนไข เช่น ช่วงวันที่ จากนั้นระบบดึงข้อมูลตามเงื่อนไข คำนวณสถิติที่สำคัญ และแสดงผลในรูปแบบกราฟและตารางเพื่อใช้ในการวิเคราะห์การใช้จ่าย

## แผนภาพกิจกรรมการส่งออกข้อมูลรายงานแบบ CSV/PDF (UC_11)

แผนภาพกิจกรรมนี้แสดงการส่งออกข้อมูลรายงาน โดยผู้ใช้งานเลือกชนิดไฟล์ที่ต้องการ จากนั้นระบบสร้างไฟล์ตามรูปแบบที่เลือก จัดเตรียมข้อมูล และส่งไฟล์ให้ผู้ใช้งานดาวน์โหลดเพื่อนำไปใช้งานต่อ

## แผนภาพกิจกรรมการตรวจสอบข้อมูลผู้ใช้งาน (UC_12)

แผนภาพกิจกรรมนี้อธิบายการทำงานของผู้ดูแลระบบในการตรวจสอบผู้ใช้งาน เริ่มจากการเข้าสู่เมนูจัดการผู้ใช้ ระบบตรวจสอบสิทธิ์แอดมินและดึงข้อมูลผู้ใช้งานเพื่อแสดงผลให้สามารถตรวจสอบรายการได้อย่างเป็นระบบ

## แผนภาพกิจกรรมการตรวจสอบงบประมาณและค่าใช้จ่ายทั้งหมด (UC_13)

แผนภาพกิจกรรมนี้แสดงการตรวจสอบข้อมูลเชิงลึกผ่านหน้าฐานข้อมูลของผู้ดูแลระบบ โดยสามารถเลือกมุมมองข้อมูลงบประมาณหรือค่าใช้จ่าย ระบบจะดึงข้อมูลและแสดงในรูปแบบตารางที่รองรับการค้นหาและการเรียงลำดับ

## แผนภาพกิจกรรมการตรวจสอบแจ้งเตือนผิดปกติ (UC_14)

แผนภาพกิจกรรมนี้อธิบายขั้นตอนการติดตามเหตุการณ์ผิดปกติ ผู้ดูแลระบบเปิดหน้าการแจ้งเตือน ระบบแสดงสถานะของแต่ละรายการ จากนั้นผู้ดูแลระบบสามารถรับทราบแจ้งเตือน และระบบจะบันทึกสถานะรวมถึงเวลาที่รับทราบ

## แผนภาพกิจกรรมการตรวจสอบบันทึกเหตุการณ์ระบบ (UC_15)

แผนภาพกิจกรรมนี้แสดงการตรวจสอบบันทึกเหตุการณ์ (Audit Log) ของผู้ดูแลระบบ โดยระบบดึงข้อมูลเหตุการณ์ทั้งหมด เรียงลำดับจากล่าสุด และแสดงรายละเอียดผู้กระทำและเวลาที่เกิดเหตุการณ์ เพื่อใช้ในการตรวจสอบย้อนหลังและกำกับดูแลระบบ
