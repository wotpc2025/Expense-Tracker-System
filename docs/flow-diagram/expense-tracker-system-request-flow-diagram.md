# Request Flow Diagram

เอกสารนี้ใช้สำหรับอธิบายลำดับการไหลของคำขอ (Request Flow) ตั้งแต่ผู้ใช้งานภายนอกจนถึงบริการแอปพลิเคชันและฐานข้อมูลของระบบ

## ภาพแผนภาพการไหลของคำขอ

1. [Request Flow Diagram (PNG)](./Request%20Flow%20Diagram.drawio.png)

## คำอธิบายโดยสรุป

1. ผู้ใช้งานส่ง HTTPS Request ผ่านโดเมนของระบบ
2. Cloudflare ตรวจสอบเส้นทางและนโยบายการเข้าถึงก่อนส่งต่อผ่าน Tunnel
3. Nginx Proxy Manager ภายใน CT101 จัดเส้นทางคำขอไปยังบริการเว็บแอป
4. แอปพลิเคชันประมวลผลคำขอและเชื่อมต่อฐานข้อมูลเพื่ออ่านหรือบันทึกข้อมูล
5. ผลลัพธ์ถูกส่งย้อนกลับตามลำดับชั้นเดิมจนถึงผู้ใช้งาน
