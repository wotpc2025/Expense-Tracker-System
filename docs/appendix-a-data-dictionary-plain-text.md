ภาคผนวก ก พจนานุกรมข้อมูล (Data Dictionary)

ภาคผนวกนี้จัดทำขึ้นเพื่อใช้อธิบายโครงสร้างข้อมูลที่ปรากฏในฐานข้อมูลของระบบ Expense Management System โดยอ้างอิงจาก ER Diagram และโครงสร้างฐานข้อมูลล่าสุดใน MySQL เพื่อให้สามารถอธิบายรายละเอียดของแต่ละตารางและแอตทริบิวต์ที่เกี่ยวข้องได้อย่างชัดเจน

ขอบเขตฐานข้อมูลในโครงงาน

โครงงานนี้มีข้อมูลผู้ใช้งานทั้งจากบริการภายนอกและฐานข้อมูลภายในระบบ ดังนี้
1. บริการภายนอก Clerk ใช้สำหรับยืนยันตัวตน (Authentication) และจัดการบัญชีผู้ใช้
2. ฐานข้อมูลภายใน MySQL ใช้จัดเก็บข้อมูลธุรกรรมและข้อมูลประกอบการทำงานของระบบ

ในเวอร์ชันล่าสุด ฐานข้อมูลภายในประกอบด้วย 5 ตารางหลัก คือ users, budgets, expenses, admin_alerts และ admin_audit_logs

หมายเหตุ: ในการทำงานจริง ระบบมีการอ้างอิงผู้ใช้งานทั้งในรูปแบบรหัสผู้ใช้ภายในระบบ (เช่น createdByUserId, actorUserId) และอีเมลของผู้ปฏิบัติการ (เช่น createdBy, actorEmail) เพื่อรองรับทั้งการเชื่อมความสัมพันธ์ของข้อมูลและการตรวจสอบย้อนหลัง

สรุปความสัมพันธ์ของข้อมูลตาม ER Diagram ล่าสุด
1. users -> budgets: ผู้ใช้หนึ่งคนสามารถมีงบประมาณได้หลายรายการ (ผ่าน createdByUserId)
2. budgets -> expenses: งบประมาณหนึ่งรายการสามารถมีค่าใช้จ่ายได้หลายรายการ (ผ่าน budgetId)
3. users -> expenses: ผู้ใช้หนึ่งคนสามารถสร้างค่าใช้จ่ายได้หลายรายการ (ผ่าน createdByUserId)
4. users -> admin_alerts: ผู้ดูแลระบบหนึ่งคนสามารถรับทราบแจ้งเตือนได้หลายรายการ (ผ่าน acknowledgedByUserId)
5. users -> admin_audit_logs: ผู้ดูแลระบบหนึ่งคนสามารถมีบันทึกเหตุการณ์ได้หลายรายการ (ผ่าน actorUserId)

หมายเหตุสำหรับการจัดทำรายงาน
1. ตาราง users ในระบบนี้เป็นตาราง profile ภายในที่ซิงก์จาก Clerk เพื่อรองรับการเชื่อมโยงข้อมูลภายในระบบ
2. การยืนยันตัวตนยังคงดำเนินการผ่าน Clerk ตามสถาปัตยกรรมระบบเดิม
3. ค่าตัวอย่างในแต่ละตารางปรับให้สอดคล้องกับข้อมูลจริงจากฐานข้อมูล ณ วันที่จัดทำเอกสาร
4. หากมีการแนบ ER Diagram ในบทที่ 3 ให้ใช้คำอธิบายความสัมพันธ์ตามรายการในหัวข้อ "สรุปความสัมพันธ์ของข้อมูลตาม ER Diagram ล่าสุด" เพื่อให้สอดคล้องกับแบบจำลองข้อมูลปัจจุบัน
