# ภาคผนวก ก พจนานุกรมข้อมูล (Data Dictionary)

ภาคผนวกนี้จัดทำขึ้นเพื่อใช้อธิบายโครงสร้างข้อมูลที่ปรากฏในฐานข้อมูลของระบบ Expense Tracker System โดยอ้างอิงจากโครงสร้างฐานข้อมูลที่กำหนดไว้ในระบบผ่าน Drizzle ORM ร่วมกับฐานข้อมูล MySQL เพื่อให้สามารถอธิบายรายละเอียดของแต่ละตารางและแอตทริบิวต์ที่เกี่ยวข้องได้อย่างชัดเจน

อนึ่ง ระบบนี้ใช้ Clerk เป็นเครื่องมือสำหรับการยืนยันตัวตนและการจัดการข้อมูลผู้ใช้งาน ดังนั้นฐานข้อมูลหลักของโครงงานจึงไม่ได้กำหนดตารางผู้ใช้งานแยกต่างหากเช่นเดียวกับระบบทั่วไป ทั้งนี้ ข้อมูลที่ใช้ระบุผู้สร้างหรือผู้ดำเนินการในบางตารางจะจัดเก็บในรูปแบบอีเมลของผู้ใช้งาน เช่นฟิลด์ createdBy และ actorEmail

## ขอบเขตความครบถ้วนของฐานข้อมูลในโครงงาน

เพื่อให้การประเมินความครบถ้วนของฐานข้อมูลเป็นไปอย่างถูกต้อง โครงงานนี้แบ่งแหล่งข้อมูลออกเป็น 2 ส่วน ได้แก่

1. ฐานข้อมูลภายในระบบ (MySQL) ซึ่งผู้พัฒนารับผิดชอบการออกแบบโดยตรง ประกอบด้วย 4 ตารางหลัก คือ budgets, expenses, admin_alerts และ admin_audit_logs
2. บริการภายนอกสำหรับยืนยันตัวตน (Clerk) ซึ่งรับผิดชอบข้อมูลบัญชีผู้ใช้ เช่น ข้อมูลสมัครสมาชิกและข้อมูลเข้าสู่ระบบ

ดังนั้น หากพิจารณาเฉพาะฐานข้อมูลภายในระบบ โครงสร้างข้อมูลถือว่า "ครบถ้วนตามขอบเขตงาน" แล้ว เนื่องจากครอบคลุมข้อมูลธุรกรรมงบประมาณ รายการค่าใช้จ่าย การติดตามสถานะการแจ้งเตือน และบันทึกเหตุการณ์ของผู้ดูแลระบบครบทุกส่วนที่ระบบหลักใช้งานจริง

## 1. ตารางข้อมูลงบประมาณ

ตารางข้อมูลงบประมาณใช้สำหรับจัดเก็บรายละเอียดของงบประมาณที่ผู้ใช้สร้างขึ้นภายในระบบ โดยประกอบด้วยข้อมูลสำคัญ ได้แก่ ชื่องบประมาณ จำนวนเงินงบประมาณ ไอคอนหรือสัญลักษณ์ประกอบ หมวดหมู่เริ่มต้น และอีเมลของผู้สร้างรายการ ดังที่แสดงในตารางที่ ก-1

**ตาราง ก-1 พจนานุกรมข้อมูลตารางข้อมูลงบประมาณ (`budgets`)**

| ชื่อ Attribute | คำอธิบาย | ชนิดของข้อมูล | ตัวอย่าง |
| --- | --- | --- | --- |
| id | รหัสเฉพาะของงบประมาณ | int | 1 |
| name | ชื่องบประมาณ | varchar(255) | ค่าอาหาร |
| amount | วงเงินงบประมาณที่กำหนด | decimal(12,2) | 3500.00 |
| icon | ไอคอนหรืออีโมจิประจำงบประมาณ | varchar(255) | 🍜 |
| category | หมวดหมู่เริ่มต้นของงบประมาณ | varchar(255) | Food |
| createdBy | อีเมลของผู้สร้างงบประมาณ | varchar(255) | user@example.com |

## 2. ตารางข้อมูลค่าใช้จ่าย

ตารางข้อมูลค่าใช้จ่ายใช้สำหรับจัดเก็บรายละเอียดของรายการค่าใช้จ่ายที่เกิดขึ้นภายใต้งบประมาณแต่ละรายการ โดยบันทึกชื่อรายการ จำนวนเงินที่ใช้จ่าย หมวดหมู่ รหัสงบประมาณที่เกี่ยวข้อง และวันเวลาที่บันทึกรายการ ดังที่แสดงในตารางที่ ก-2

**ตาราง ก-2 พจนานุกรมข้อมูลตารางข้อมูลค่าใช้จ่าย (`expenses`)**

| ชื่อ Attribute | คำอธิบาย | ชนิดของข้อมูล | ตัวอย่าง |
| --- | --- | --- | --- |
| id | รหัสเฉพาะของรายการค่าใช้จ่าย | int | 15 |
| name | ชื่อรายการค่าใช้จ่าย | varchar(255) | กาแฟ |
| amount | จำนวนเงินที่ใช้จ่าย | decimal(12,2) | 65.00 |
| budgetId | รหัสงบประมาณที่รายการนี้สังกัด | int | 1 |
| category | หมวดหมู่ของค่าใช้จ่าย | varchar(255) | Food |
| createdAt | วันและเวลาที่บันทึกรายการ | timestamp | 2026-01-14 08:30:00 |

## 3. ตารางข้อมูลการแจ้งเตือนผู้ดูแลระบบ

ตารางข้อมูลการแจ้งเตือนผู้ดูแลระบบใช้สำหรับจัดเก็บรายละเอียดของเหตุการณ์แจ้งเตือนที่เกี่ยวข้องกับการติดตามและการดูแลระบบ โดยมีข้อมูลสำคัญ เช่น รหัสการแจ้งเตือน สถานะการแจ้งเตือน วันเวลาที่รับทราบ และผู้ที่รับทราบการแจ้งเตือน ดังที่แสดงในตารางที่ ก-3

**ตาราง ก-3 พจนานุกรมข้อมูลตารางข้อมูลการแจ้งเตือนผู้ดูแลระบบ (`admin_alerts`)**

| ชื่อ Attribute | คำอธิบาย | ชนิดของข้อมูล | ตัวอย่าง |
| --- | --- | --- | --- |
| id | รหัสเฉพาะของการแจ้งเตือน | int | 3 |
| alertKey | รหัสอ้างอิงหรือชื่อคีย์ของการแจ้งเตือน | varchar(255) | duplicate-expense-detected |
| status | สถานะของการแจ้งเตือน | varchar(255) | acknowledged |
| acknowledgedAt | วันและเวลาที่มีการรับทราบการแจ้งเตือน | timestamp | 2026-02-01 10:15:00 |
| acknowledgedBy | อีเมลของผู้ดูแลระบบที่รับทราบการแจ้งเตือน | varchar(255) | admin@example.com |

## 4. ตารางข้อมูลบันทึกการทำงานของผู้ดูแลระบบ

ตารางข้อมูลบันทึกการทำงานของผู้ดูแลระบบใช้สำหรับจัดเก็บประวัติการดำเนินการต่าง ๆ ที่เกิดขึ้นภายในระบบ เช่น การเพิ่ม การแก้ไข หรือการลบข้อมูล เพื่อประโยชน์ในการตรวจสอบย้อนหลังและการติดตามการใช้งานของผู้ดูแลระบบ ดังที่แสดงในตารางที่ ก-4

**ตาราง ก-4 พจนานุกรมข้อมูลตารางข้อมูลบันทึกการทำงานของผู้ดูแลระบบ (`admin_audit_logs`)**

| ชื่อ Attribute | คำอธิบาย | ชนิดของข้อมูล | ตัวอย่าง |
| --- | --- | --- | --- |
| id | รหัสเฉพาะของบันทึกเหตุการณ์ | int | 12 |
| action | ชื่อการกระทำที่เกิดขึ้นในระบบ | varchar(255) | create_budget |
| targetType | ประเภทข้อมูลที่ถูกดำเนินการ | varchar(255) | budget |
| targetCount | จำนวนรายการที่ได้รับผลกระทบ | int | 1 |
| message | รายละเอียดข้อความของเหตุการณ์ | varchar(500) | Created budget ค่าอาหาร |
| actorEmail | อีเมลของผู้ดูแลระบบหรือผู้ใช้งานที่เป็นผู้กระทำ | varchar(255) | admin@example.com |
| createdAt | วันและเวลาที่บันทึกเหตุการณ์ | timestamp | 2026-02-10 14:22:31 |

## สรุปความสัมพันธ์ของข้อมูล

จากโครงสร้างฐานข้อมูลของระบบ พบว่าตาราง budgets และ expenses มีความสัมพันธ์แบบหนึ่งต่อหลาย โดยข้อมูลงบประมาณ 1 รายการสามารถเชื่อมโยงกับข้อมูลค่าใช้จ่ายได้หลายรายการผ่านฟิลด์ budgetId ในตาราง expenses ส่วนตาราง admin_alerts และ admin_audit_logs เป็นตารางที่ใช้สนับสนุนการบริหารจัดการระบบ การติดตามสถานะการแจ้งเตือน และการตรวจสอบเหตุการณ์ย้อนหลัง

## หมายเหตุสำหรับการจัดทำรายงาน

1. ในการจัดพิมพ์ลงในรูปเล่มรายงาน สามารถใช้ชื่อภาษาไทยของแต่ละตารางเพื่อให้อ่านเข้าใจง่ายและสอดคล้องกับบริบทของโครงงาน
2. หากต้องการนำเสนอเฉพาะตารางธุรกรรมหลักของระบบ สามารถเลือกอธิบายเฉพาะตาราง budgets และ expenses ได้ โดยย้ายรายละเอียดของตาราง admin_alerts และ admin_audit_logs ไปไว้ในภาคผนวกเพิ่มเติม
3. หากมีการแนบแผนภาพความสัมพันธ์ของข้อมูล ควรเน้นความสัมพันธ์ระหว่างฟิลด์ id ของตาราง budgets กับฟิลด์ budgetId ของตาราง expenses ซึ่งเป็นความสัมพันธ์เชิงอ้างอิงหลักของฐานข้อมูลในระบบนี้

## คำอธิบายเพิ่มเติมเกี่ยวกับฟิลด์วันและเวลา

ในระบบฐานข้อมูลโดยทั่วไป ฟิลด์ที่เกี่ยวข้องกับวันและเวลา เช่น createdAt และ updatedAt มักถูกใช้เพื่อบันทึกเวลาที่มีการสร้างข้อมูลและเวลาที่มีการปรับปรุงข้อมูลล่าสุด อย่างไรก็ตาม การกำหนดฟิลด์ดังกล่าวไม่จำเป็นต้องมีอยู่ในทุกตารางเสมอไป แต่ควรพิจารณาตามลักษณะการใช้งานของข้อมูลในแต่ละตารางเป็นสำคัญ

สำหรับโครงงานนี้ มีการกำหนดฟิลด์วันและเวลาเฉพาะในตารางที่จำเป็นต่อการติดตามธุรกรรมหรือการตรวจสอบย้อนหลัง เช่น ตาราง expenses และตาราง admin_audit_logs ซึ่งต้องอาศัยข้อมูลเวลาในการอธิบายลำดับเหตุการณ์ของการใช้งานระบบ ขณะที่ตาราง budgets ไม่ได้กำหนดฟิลด์ createdAt และ updatedAt เนื่องจากโครงสร้างปัจจุบันของระบบยังไม่ได้ใช้ข้อมูลเวลาดังกล่าวโดยตรงในการประมวลผลหรือแสดงผล ส่วนตาราง admin_alerts ใช้ฟิลด์ acknowledgedAt เพื่อระบุเวลาที่ผู้ดูแลระบบรับทราบการแจ้งเตือน ซึ่งสอดคล้องกับวัตถุประสงค์ของตารางมากกว่าการบันทึกเวลาแบบทั่วไป

ดังนั้น การที่บางตารางไม่มีฟิลด์ createdAt และ updatedAt ไม่ถือว่าเป็นข้อผิดพลาดของการออกแบบฐานข้อมูล หากแต่เป็นการออกแบบให้เหมาะสมกับการใช้งานจริงของระบบ และช่วยลดความซ้ำซ้อนของข้อมูลที่ไม่จำเป็น ทั้งนี้ หากในอนาคตมีการขยายความสามารถของระบบ เช่น การติดตามประวัติการแก้ไขงบประมาณ หรือการตรวจสอบเวลาการสร้างการแจ้งเตือนเพิ่มเติม ก็สามารถพิจารณาเพิ่มฟิลด์ดังกล่าวในตารางที่เกี่ยวข้องได้ตามความเหมาะสม
