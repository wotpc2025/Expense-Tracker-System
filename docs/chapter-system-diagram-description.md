# ข้อความบรรยายใต้รูป System Diagram

ภาพที่ 3-1 รูปภาพแผนภาพสถาปัตยกรรมระบบบริหารการจัดการค่าใช้จ่าย (System Diagram)

จากภาพที่ 3-1 ระบบบริหารการจัดการค่าใช้จ่ายถูกออกแบบในลักษณะ Web Application โดยผู้ใช้งานทั่วไปและผู้ดูแลระบบเข้าใช้งานผ่านเว็บเบราว์เซอร์ จากนั้นคำขอทั้งหมดจะถูกส่งไปยังแอปพลิเคชันหลักที่พัฒนาด้วย Next.js ซึ่งทำหน้าที่ทั้งฝั่งส่วนติดต่อผู้ใช้และฝั่งประมวลผลเซิร์ฟเวอร์

ในส่วนของการยืนยันตัวตน ระบบเชื่อมต่อกับบริการภายนอก Clerk เพื่อจัดการกระบวนการสมัครสมาชิก เข้าสู่ระบบ และตรวจสอบสถานะเซสชันผู้ใช้งาน ขณะเดียวกันข้อมูลธุรกรรมหลักของระบบจะถูกจัดเก็บในฐานข้อมูล MySQL ผ่าน Drizzle ORM โดยประกอบด้วยตารางหลัก ได้แก่ users, budgets, expenses, admin_alerts และ admin_audit_logs

นอกจากนี้ ระบบยังเชื่อมต่อกับบริการ OpenRouter AI สำหรับฟีเจอร์สแกนใบเสร็จ โดยแอปพลิเคชันจะส่งข้อมูลรูปภาพใบเสร็จไปประมวลผลและรับผลลัพธ์กลับมาในรูปแบบรายการค่าใช้จ่ายเพื่อให้ผู้ใช้ตรวจสอบก่อนบันทึกเข้าสู่ฐานข้อมูล

ดังนั้น แผนภาพนี้สะท้อนให้เห็นความสัมพันธ์ขององค์ประกอบหลักของระบบอย่างครบถ้วน ทั้งส่วนผู้ใช้งาน แอปพลิเคชัน บริการยืนยันตัวตน บริการ AI ภายนอก และฐานข้อมูล ซึ่งทำงานร่วมกันเพื่อรองรับการจัดการค่าใช้จ่ายอย่างเป็นระบบและตรวจสอบได้

## เวอร์ชันสั้นสำหรับใส่ใต้รูป (ถ้าต้องการแบบกระชับ)

ภาพที่ 3-1 แสดงโครงสร้างการทำงานของระบบบริหารการจัดการค่าใช้จ่าย โดยผู้ใช้งานเข้าผ่านเว็บเบราว์เซอร์ไปยังแอปพลิเคชัน Next.js ซึ่งเชื่อมต่อ Clerk สำหรับยืนยันตัวตน เชื่อมต่อ MySQL ผ่าน Drizzle ORM สำหรับจัดเก็บข้อมูลหลัก และเชื่อมต่อ OpenRouter AI สำหรับประมวลผลสแกนใบเสร็จ
